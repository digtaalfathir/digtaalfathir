# Personal Brand Roadmap — Rifky Andigta

**Systems & Automation Engineer**
_I build systems that connect machines, software, and people._

> Ini bukan roadmap belajar, bukan roadmap coding.
> Ini roadmap membangun nama **"Rifky Andigta"**.
>
> Target: **3 tahun dari sekarang**. Kalau orang Googling _"Rifky Andigta"_,
> yang muncul bukan cuma sosial media — tapi GitHub, Website, npm, LinkedIn,
> Blog, Dokumentasi, Library, dan Produk. Semuanya konsisten.

---

## Kenapa "Systems & Automation Engineer"

Title ini dipilih karena:

- ✅ Tidak mengunci di embedded
- ✅ Tidak mengunci di web
- ✅ Tidak mengunci di manufaktur
- ✅ Tetap relevan untuk AI, Cloud, Desktop, IoT, Robotics
- ✅ Tetap cocok kalau nanti mendirikan perusahaan sendiri

Yang terpenting, title ini sesuai arah semua project: **menghubungkan banyak
sistem menjadi satu solusi.**

## The 10 Phases

### Phase 1 — Identity _(1–2 minggu)_
Tujuan: dalam 10 detik buka GitHub, orang langsung tahu _"Oh, dia engineer automation."_

- Header: nama + title + one-liner
- Bio fokus: Automation · Monitoring · Embedded · Industrial Communication · Developer Tools · Security
- Profile README premium: About · Projects · Tech Stack · Open Source · Currently Building · Roadmap · Contact

### Phase 2 — Bersihkan GitHub
Dari `test123`, `coba`, `printerbaru` → jadi struktur rapi:

```
applications/   libraries/   firmware/   examples/   archive/
```

Setiap repo wajib: ✅ README ✅ Screenshot ✅ Installation ✅ License ✅ Topics

### Phase 3 — Tentukan Flagship Project
Jangan 40 project. Cukup **5 unggulan**, sisanya pendukung.

- ⭐ Autopath — visual RPA workflow designer
- ⭐ Twinscape — web ops console: 2D/3D monitoring + remote SSH/VNC
- ⭐ Protokit — device comms libraries (Modbus, MC, Open Protocol) → npm
- ⭐ Pinstream — ESP32 GPIO I/O streamed over WebSocket
- ⭐ BayRecon — ESP8266 WiFi security toolkit (research/lab)

### Phase 4 — Build Ecosystem
Apps → Libraries → Firmware → Documentation. Semuanya saling memakai.

```
Autopath · Twinscape
     ↓  consume
@digta/modbus · @digta/mcprotocol · @digta/openprotocol   (Protokit)
```

### Phase 5 — npm
Mulai jadi **maintainer**, bukan cuma programmer.

```
@digta/modbus   v0.1 → v0.2 → v1.0 → v2.0
```

Tiap package punya: Release · Changelog · Issue · Discussion · Roadmap

### Phase 6 — Documentation
Yang membedakan engineer biasa dengan maintainer.

`docs.digta.dev` → Getting Started · Tutorial · API · Examples · FAQ

### Phase 7 — Website
Bukan portfolio, tapi **landing page**.

```
Rifky Andigta — Systems & Automation Engineer
Building developer tools, automation platforms, and connected systems.
Projects → Libraries → Articles → Contact
```

### Phase 8 — LinkedIn
**1 posting per minggu.** Tidak usah viral, yang penting konsisten.

> "Hari ini berhasil membuat simulator Open Protocol. Kenapa simulator penting? ... GitHub: ..."

### Phase 9 — Technical Writing
Wajib. Supaya orang Googling bisa menemukanmu.

- How Open Protocol Works
- Building Factory Monitoring
- Printer Middleware
- Electron for Industrial Apps
- ESP32 Communication
- TCP Heartbeat
- Industrial SDK

### Phase 10 — Community
GitHub Discussion · Discord · Telegram · Issue · Pull Request.
Sekarang bukan cuma programmer — tapi **maintainer**.

## Prinsip

Konsistensi > viral. Orang mengingat engineer bukan karena satu project viral,
tapi karena **arah karyanya selalu jelas**. Setiap commit, repository, dan
postingan harus memperkuat branding yang sama.
